# 문제 파일 작성 규칙

## 템플릿 구조

`Problems/TEMPLATE.md`를 복사해서 시작한다.

```
---
date: YYYY-MM-DD
type: exam | practice
---

# 제목

## 문제          ← 배경 설명 + 예시 코드 + 질문 목록
## Q1. 질문 제목  ← 내 답변 → 평가 → 이상적인 답변
## Q2. 질문 제목
...
```

---

## Frontmatter 규칙

| 필드 | 값 | 설명 |
|------|----|------|
| `date` | `YYYY-MM-DD` | 필기 시험 또는 작성 날짜 |
| `type` | `exam` \| `practice` | 실제 시험 복기 / 연습 문제 |

---

## 각 질문 작성 순서

1. **질문** — `## Q1. 질문 제목` 헤딩 바로 아래에 `**질문:**` 항목으로 질문 전문을 재기재
   - 스크롤 없이 질문·답변을 함께 볼 수 있도록 하기 위함
2. **내 답변** — 당시 작성한 답변을 매끄럽게 다듬어 기록
3. **평가** — 인용 블록(`>`)으로 작성
4. **이상적인 답변** — 놓친 부분을 포함한 완전한 답변

### 평가 기호

| 기호 | 의미 |
|------|------|
| `✅` | 정확 |
| `⚠️` | 부분 정확 (핵심은 맞으나 누락 있음) |
| `❌` | 오답 |

---

## 코드 블록 규칙

- 언어 태그 항상 명시: ` ```csharp `
- 유니티 코드는 Unity 패키지 없이 실행 불가이므로 `.md` 코드 블록으로만 기록
- `Program.cs`는 순수 C# 로직 실험 전용

## 네이밍 규칙

| 구분 | 경로 | 파일명 |
|------|------|--------|
| Private | `Problems/Private/[회사명]/` | `N번 문제.md` |
| Public | `Problems/Public/` | `YYYY-MM-DD_제목.md` |

**Public 파일명 주의:**
- 언어명(C#, Python 등) 미포함 — 이 프로젝트는 C# 전용이므로 불필요
- 예) `2026-04-19_비효율-코드-패턴.md` ✅ / `2026-04-19_C#-비효율-코드-패턴.md` ❌
